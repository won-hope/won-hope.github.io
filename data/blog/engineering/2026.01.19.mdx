## 👶 Introduction

**“우리 도현이 저번 예방접종 언제 맞았지?”  
“여보, 퇴근할 때 도현이 기저귀랑 우유 좀 사 와.”**

아이를 키우기 시작하면서, 가족의 데이터와 태스크는 생각보다 빠르게 흩어지기 시작했다.  
예방접종 기록은 병원 앱에, 수유 기록은 육아 앱에, 가계부는 엑셀에, 일정은 캘린더에.

특히 와이프가 부탁하는 심부름은 카카오톡 공지사항에 띄워두곤 하는데,  
퇴근길 바쁜 와중에 카톡을 열어 다시 확인하는 일은 생각보다 번거롭다.

문제는 명확했다.

- 🧩 **데이터 파편화:** 육아 기록, 가계부, 일정이 여러 서비스에 흩어져 있음
- 🔔 **알림의 휘발성:** 카톡 공지는 쉽게 묻히고, 중요한 심부름은 놓치기 쉬움
- 🚫 **통제권 부재:** 정작 **내 가족의 데이터와 운영 프로세스에 대한 통제권은 내가 갖고 있지 않음**

결론은 단순했다.  
와이프와 나, 그리고 우리 아기 도현이를 위한  
**Private Family ERP — Project OHANA**를 직접 만들기로 했다.

단순한 육아 기록 앱이 아니라,  
**가족 데이터를 직접 소유하고 운영하는 작은 ERP**를 목표로 삼았다.  
그리고 여기에 ‘가족 전용 심부름 알림’ 기능을 핵심 기능으로 포함했다.

---

### 🎯 핵심 목표

1. **데이터 주권:** 최종 저장소는 **Google Sheets (내 계정)**
2. **사용성:** 한 손으로 입력 가능한 **One-Tap UI** + 즉각적인 **Push 알림**
3. **유지보수:** 서버 운영 부담 **Zero (Serverless)**

## 🏗️ Body

### 1. The Core Philosophy: "No Vendor Lock-in"

이 프로젝트의 제1원칙은 단 하나다. **데이터 소유권.**

앱이 사라져도, 서비스가 종료되어도,  
우리 가족의 모든 기록은 `xlsx` 파일로 남아야 한다.

그래서 구조를 의도적으로 단순하게 설계했다.

- **Logic:** 앱은 입력과 조회만 담당한다. 비즈니스 로직은 최소화한다.
- **Storage:** 모든 최종 데이터는 **Google Sheets**에 저장한다.
- **Task Management:** 심부름 등록 시 즉시 알림을 보내고, 결과는 시트에 자동 아카이빙한다.

특정 벤더나 서비스에 데이터를 맡기지 않는다.  
**데이터의 최종 소유자는 항상 사용자 자신이다.**

---

### 2. UX Strategy: "One-Handed & Don't Forget the Errand"

육아 중에는 기억력이 자산이다. 하지만 현실은 가혹하다.  
한 손은 아이를 안고 있고, 다른 한 손으로 **1초 안에 기록해야 한다.**

기존 육아 앱의  
`메뉴 → 진입 → 선택 → 시간 설정 → 저장`  
플로우는 현실 육아 환경에서는 과하다.

**Project OHANA**의 UX 목표는 극단적으로 단순하다.

- **One-Tap Request:** 수행자 선택 + 항목 입력 → 즉시 등록
- **Smart Notification:** 등록 즉시 수행자 폰 상단 알림 바에 고정 표시
- **Fast Logging:** 버튼 하나로 **0.1초 내 기록 완료**, 상세 수정은 나중에 처리

### 3. Architecture: The 2026 Hybrid Stack

이 프로젝트의 핵심 전략은 명확하다.

> **“클라이언트는 가볍게, 동기화는 Supabase에, 저장은 Google에.”**

백엔드 운영 부담을 없애기 위해  
**Supabase Edge Functions + Expo Push API + Google Sheets** 구조를 채택했다.

#### 🛠️ Tech Stack

- 📱 **App Framework: Expo SDK (Latest)**
  - React Native New Architecture 기반
  - `Expo Router`, `NativeWind` 사용

- ⚡ **Sync Engine: Supabase**
  - 부부 간 실시간 동기화 전용 레이어
  - 영구 저장소가 아닌 **중간 버퍼 + 이벤트 트리거 역할**

- 🔔 **Notification: Expo Push Notification**
  - 심부름 등록 시 즉시 수행자에게 알림

- 📊 **Data Warehouse: Google Sheets & Calendar**
  - Edge Function 트리거 → Google Sheets API로 Row 추가
  - 일정 데이터는 **Google Calendar 자체를 DB처럼 사용**

Supabase는 빠르고 편리하지만,  
**데이터의 최종 목적지는 항상 Google이다.**
